# DESIGN_DIRECTION.md

## Product
JS Auto Parts advanced website design exploration

## Current Stage
Design-first exploration before formal product planning.

This is not a Codex-ready implementation handoff. The goal is to explore a premium, high-impact visual direction for a responsive website before PRD, MVP scope, content structure, and implementation planning are finalized.

## Target User
- Auto-rickshaw owners
- Auto-rickshaw drivers
- Bike owners
- Scooty owners
- Local mechanics
- Local customers looking for tyres, helmets, oils, batteries, lights, horns, accessories, and spare parts

## Product Feeling
Premium, powerful, trusted, local, mechanical, energetic, fast, road-ready.

The site should feel like a modern automotive garage, not a generic parts catalogue. It should make a local shop feel professional, established, and easy to contact.

## Design Goal
Create a visually advanced, high-impact automotive website that helps customers quickly trust JS Auto Parts, understand product availability, and contact the shop by call or WhatsApp.

Primary conversion goal:
- Call enquiry
- WhatsApp enquiry

Secondary goal:
- Request part / quote form

## Core Design Idea
“Khordha’s trusted auto parts shop, redesigned like a premium garage.”

The design should combine:
- Real local trust
- Dense product availability
- Premium dark automotive visuals
- Fast call / WhatsApp action
- Clear category discovery

## Visual Style
- Dark metallic base
- Tyre rubber textures
- Chrome / steel accents
- Electric blue light streaks
- Amber/yellow CTA highlights
- Cinematic garage lighting
- Subtle mechanical animation
- High contrast product cards
- Local trust details clearly visible

## Brand Personality
JS Auto Parts should feel:
- Experienced
- Reliable
- Stocked
- Road-focused
- Local and accessible
- Bold, but not flashy
- Premium, but not distant

## Color Direction
Primary background:
- Carbon black
- Deep graphite
- Dark tyre-rubber texture

Primary accent:
- Electric blue for motion, highlights, active states, and hero glow

CTA accent:
- Amber/yellow for Call and Request Quote
- WhatsApp green for WhatsApp-specific actions only

Support colors:
- Chrome silver
- Steel grey
- Warm garage light
- Brake red for error states only
- Success green for confirmation states

## Typography Direction
Headings:
- Bold, condensed, mechanical-feeling sans serif
- Strong uppercase or title case for hero and section headers

Body:
- Highly readable modern sans serif
- Avoid overly futuristic text that hurts readability

Labels:
- Compact, clear, high contrast
- Use small technical labels like “Tyres”, “Batteries”, “Engine Oil”, “Helmets”

Buttons:
- Heavy-weight, high-contrast, tactile
- Should feel like solid physical switches or dashboard controls

## Layout Direction
Desktop:
- Wide cinematic hero
- Sticky header
- Two-column hero: copy + animated tyre/product visual
- Category card grid
- Product visual showcase with parallax shelves
- Strong CTA blocks
- Footer with address, hours, phone, WhatsApp

Mobile:
- Mobile-first hierarchy
- Sticky top header or compact header
- Bottom Call / WhatsApp action bar
- Short hero
- Large category cards
- No dense desktop grids
- Thumb-friendly CTAs
- Contact information visible without hunting

## Photo Usage
Use uploaded shop photos as reference for:
- Product density
- Local inventory
- Shop atmosphere
- Real categories
- Shelving and accessories
- Tyres, helmets, oils, batteries, lights, horns

Do not use raw shop photos as the main hero unless heavily edited. Prefer a stylized/generated hero inspired by the photos:
- Tyre in foreground
- Product shelves behind
- Blue light streaks
- Amber garage light
- Metallic/rubber textures
- Slight motion blur

## Animation Direction
- Tyre spin entrance in hero
- Light streaks behind hero
- Parallax product shelves
- Product cards with subtle 3D hover tilt
- Floating WhatsApp pulse
- Category cards reveal on scroll
- Brand logo marquee
- 25+ years counter animation
- Smooth page transitions
- Mechanical loading animation using gear/tyre style

## Accessibility Notes
- Maintain strong contrast on dark backgrounds
- Do not place small grey text on black
- CTA buttons must be large and readable
- Mobile tap targets should be at least 44px high
- Do not rely on color alone for status
- Support reduced-motion mode
- Make phone and WhatsApp actions clear with text labels, not icons alone

## States to Design
- Loading state for animated product/category sections
- Product search empty state
- Quote form success state
- Quote form error state
- Disabled button state
- Mobile menu open state
- Reduced-motion state
- Offline/contact fallback message if form cannot send

## What to Avoid
- Shopping cart
- Online payment
- Login
- Customer account
- Delivery tracking
- Admin dashboard
- Mechanic booking
- Fake Google map
- Fake delivery service
- Fake reviews
- Generic SaaS look
- Overly clean white template style
- Random neon colors
- Heavy animation that blocks contacting the shop
- Hiding phone/WhatsApp below the fold
- Desktop layout copied directly into mobile
